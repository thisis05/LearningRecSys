Matrix Factorization

Matrix Factorization이란 협업 필터링 모델의 일종인 Latent factor model

유저의 아이템에 대한 Rating information을 담고 있는 Rating Matrix (Mx N)을 SVD (Sigular Vector Decomposition)을 통해 MxF, NxF 차원의 행렬로 분해하는 원리

유저와 아이템을 F차원의 잠재공간으로 매핑한 뒤 두 Matrix를 dot product하여 나타낼 수 있으므로 이 때 나타나는 MxF, NxF Matrix (이를 Latent factor라고 함) 학습시켜 새로운 Item에 대한 기존 User의 Rating을 예측하는 등의 추천 업무를 진행할 수 있음

그러나, 이 모델의 경우 Task Specific하여 General Data Set에 적용하기 힘들다는 단점이 있음

따라서 좀더 일반적인 모델로 발전한 것이 Factorization Machine이라고 볼 수 있음

🔭 Factorization Machine

FM은 일종의 Polynomial Regression에 가까운 모습을 띈다고 볼 수 있음

MF와 다른 점은 User, Item에 대한 Spare Matrix 뿐만 아니라 Time, Last Rated 등 다양한 Feature를 사용한다는 점에 있음

일종의 간단한 다항회귀식이라고 보면된다. 하나의 row를 독립변수 1개로 보아 회귀식을 전개하고, 각 Row간의 연관성이 있는지 파악하기 위해 다음과 같은 모델을 세웠다고 보면된다.

그러나, row i와 row j간의 연산을 진행하기에 계산량이 기하급수적으로 발생하게 된다. 이를 해결하기 위해서 FM에서는 행렬 분해를 사용한다

  1. 먼저 각 Feature를 F차원의 Matrix로 매핑한다.

    예를 들어 만약 Row 1의 User가 6차원이고, Item이 7차원인 one-hot encoding verctor라면 이를 Feature 별로 F차원으로 매핑하므로, 6 x F + 7 x F + 다른 특성의 차원개수 x F … 가 되는 것이다.

  1. WWW﻿는 wijwijwij﻿ 로 이루어진 행렬이다.

    이는 대칭행렬이다. 또한 이는 positive semi-definite 특성을 띈다.

    따라서 이는 Local Minimun = global Minimum이 되며 VVTVV^TVVT﻿로 분해가 가능하다. 그렇기에 기존에는 WWW﻿ 전체를 최적화시켜야 했다면 이 원리를 이용해 VVV﻿만 최적화 시키면 된다.

    계산량을 급격하게 줄일 수 있는 것이다.

따라서, 최종적인 식은 다음과 같이 표현된다.

우리는 따라서 w0,vi,f,wiw_0, v_i,_f, w_iw0​,vi​,f​,wi​﻿만 학습시키면 되는 것이다.

✅ Importance of FM

기존 RecSys 모델들은 Sparse한 데이터가 대량으로 들어올 경우, SVM 적용이 어려워 성능이 떨어지는 모습을 보였다. 그러나 이러한 FM의 등장으로 계산량의 감소를 통해 다음과 같은 성능 향상을 볼 수 있었다.

두 번째로, 이 Model은 Task Specific하지 않고 General하게 적용이 가능하다.
